TensorFlow - Distributed Computing

  • Learn
  • Practice

In production, our models get complex and our datasets too. It is not possible to process them all on a single machine. We'll have to get multiple machines and use them all together to get our work done. In this tutorial, we'll learn to use distributed system for TensorFlow. For better understanding we have broken it down into steps. Follow step by step.

Step 1 − As always, Import the necessary modules. 

import tensorflow as tf

 

Step 2 − Now let's Create a TensorFlow cluster with one node. This node will be responsible for a job that that has name "hackersfriend_worker" and that will operate at localhost:2222.

cluster_spec = tf.train.ClusterSpec({'hackersfriend_worker' : ['localhost:2222']})
server = tf.train.Server(cluster_spec)
server.target

You'll get an output like this:

'grpc://localhost:2222'
The server is currently running.

 

Step 3 − We can calculate server configuration with respective session by executing this command −

server.server_def

on running this command, you should get following output:

cluster {
   job {
      name: "hackersfriend_worker"
      tasks {
         value: "localhost:2222"
      }
   }
}
job_name: "hackersfriend_worker"
protocol: "grpc"

 

Step 4 − Launch a TensorFlow session with the execution engine being the server. Use TensorFlow to create a local server and use lsof to find out the location of the server.

ess = tf.Session(target = server.target)
server = tf.train.Server.create_local_server()

 

Step 5 − View devices available in this session.

devices = sess.list_devices()

 

Step 6 - Finally close the session.

for d in devices:
   print(d.name)
sess.close()
